            ==========================================
                        Hardware Registers
            ==========================================
So you might be asking yourself....what is the purpose of DDRB = 0b00000001?
Almost all of the pins on the AVR chips are configurable as either input or output pins. That is,
they can be internally hooked up as either voltage-sensing components or as transistor switches that
will connect the pin through to the VCC or GND voltage levels.

Each slot in the AVR's memory is a byte, and each byte has eight bits. For a "normal" RAM location, these
bits are used to encode(in binary) the number you'd like to store.
Inside the chip, each bit in RAM is hooked up to a network of transistors that keep whatever electrical state
was last imposed on them.
The ATMEGA328p has 2K bytes of internal SRAM(so rouhgly 16384bits)

When you want to configure a microcontroller pin, it looks the same in code as saving a value into a variable.
The chip's hardware registers are just like the RAM storage slots that you use for variables, only they each have side
effects.
So when you write DDRB = 0b00000001, you are not just storing the number one in a memory location, you are also flipping
the transistor switches that are directly wired to each of the eight memory cells.

Inside the AVR chip, the data-direction register(DDR) memory locations are physically wired up to the transistor switches that
determine whether a given pin is in input or output mode.
When you write the numeric value one to that register, it is remembered....
    --When a one is stored in that register bit, the output switch is closed, thus pin configured as output
    --When a one is stored in that register bit, the input switch is the one closed and the other open, thus pin configureed as 
    input
Because the hardware registers are physically connected to the input/output circuitry, the location CANNOT CHANGE(i mean just 
look at atmega328p datasheet the pins from one to 32 each have their  own designated spot on the pin!)
These registers are typically spelled out in the io.h header file so that we do not have to remember their numerical value, but
 rather mnemonic names like DDRB, for "data-direciton register B"

So try to think of each register as being a row of eight switches, and writing a number into that register is the equivalent of
flipping some of the switches.
            ==========================================
                        Three most important hardware registers
            ==========================================
Each bank of pins(B,C and D on the Mega series AVRS) has three hardware register memory locations associated with it. Let x stand 
for each bank'S letter: ex) DDRB, DDRC, DDRD
    --DDRx data-direction registers(port x): These registers control whether each pin is configured for input or output-hence the 
    data direction. After a reset or power-up, the default state is all zeros, corresponding to the pins being configured for 
    input. TO enable the pin as an output just toggle the bit for the slot
    
    --PORTx port-x-data registers: When the DDRx bits are set to one for a given pin, the PORT register controls whether that pin 
    is set to logic high or low(VCC voltage or ground)

    With the DDR configured for input, setting the PORT bits on a pin will control whether it has internal pull-up resistor attached
     or whether it is a  high-impendance state, effectively disconnected from the circuit....
    
    --PINx port-x-input pins address: The PIN register addresses are where you read the digital voltage values for each pin that 
    is configured as input. Each PINx memory location is hooked up to a comparator circuit allowing you to only check if the pin
     is voltage high or low(reads voltage values!) reading like a normal variable.

!!!!A REMINDER: Always remember to set up your DDRx pins to know which pins are going to be outputs or inputs. Not only that 
KNOW which register is for which...
    --DDRx for configuration
    --PORTx for signaling output or configuring internal resistors
    --PINx for reading values of input pins
